# 13 — Operation REPORT Draft / Черновик операции «Отчёт»

Этот файл — черновик будущей операции `REPORT`.  
Важно: не считать всё ниже реализованным кодом.

---

## 1. Статус

```text
REPORT ещё не реализован
```

Планируемое ТЗ следующего этапа — **ТЗ-10** (операция «Отчёт» / REPORT); детали появятся в отдельном постановочном файле.

Документ нужен, чтобы Cursor понимал направление, но не принимал будущие правила за текущий код.

---

## 2. Назначение

`REPORT` должен фиксировать расход подотчётных средств, начислять заработанное и влиять на аналитику проекта.

---

## 3. Предварительные участники

Возможные участники процесса:

```text
Инициатор: EMPLOYEE / PARTNER / PROJECT_HEAD
Технадзор: SUPERVISOR, если назначен
Руководитель проекта: PROJECT_HEAD
Заказчик: CUSTOMER
```

---

## 4. Предварительный lifecycle

Черновая цепочка:

```text
CREATED
→ SUPERVISOR_APPROVAL, если нужен
→ PROJECT_HEAD_APPROVAL
→ CUSTOMER_APPROVAL
→ WAITING_24_HOURS
→ COMPLETED
```

Точные статусы ещё нужно согласовать отдельно.

---

## 5. Финансовая логика пока не финализирована

Предварительно отчёт будет:

```text
уменьшать accountable_balance исполнителя
увеличивать accountable_spent
увеличивать personal_earned получателей
влиять на расход проекта
влиять на задолженность
```

Но точная математика должна быть отдельным ТЗ.

---

## 6. Связь с переводами

Отчёт может создавать или подготавливать связанные операции `TRANSFER`.

Важно:

```text
переводы живут своим lifecycle
откат отчёта не должен автоматически ломать lifecycle переводов без отдельного правила
```

## 8. Связь со статьями расходов (ТЗ-10A)

Канон по справочнику статей расходов проекта — **`docs/10_operations/14_PROJECT_EXPENSE_ITEMS.md`**. При проектировании и реализации **REPORT** нужно опираться на этот документ (структура долей, `counterparty_id`, soft-delete).

```text
статьи расходов описаны в 14_PROJECT_EXPENSE_ITEMS.md
REPORT использует статьи расходов согласно этому модулю
soft-deleted статья не должна ломать уже существующие отчёты (исторические ссылки)
финансовые дельты REPORT в продукте пока не реализованы
```
