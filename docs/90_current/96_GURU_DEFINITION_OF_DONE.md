# 96 — Definition of Done (GURU, для Cursor и ревью)

Чеклист перед merge в **main**. Детали коммитов — `docs/90_current/93_GURU_GIT_COMMIT_STANDARD.md`.

---

## Правила

1. **Код** соответствует канону в `docs/README_MODULAR.md` и релевантным файлам `docs/00_core/`, `docs/10_operations/`, `docs/20_api/`, `docs/30_flutter/`.
2. **Бизнес-логика** не живёт в контроллере: контроллер — запрос/ответ и вызов сервисов.
3. **Права доступа** — через `*AccessService` / `*VisibilityService` / middleware воркспейса, а не разрозненные копии условий в UI/backend без причины.
4. **Финансовая математика** проводится только в `TransferBalanceService`, `IncomeBalanceService` и согласованных wallet-сервисах; не в контроллере, модели и не в Flutter.
5. **Lifecycle статусов** — только в `TransferLifecycleService` / `IncomeLifecycleService` (или будущих `*LifecycleService` новых операций).
6. **Flutter** не считает деньги и не эмулирует переходы статусов; отображает данные и действия с API.
7. **Документация** обновлена, если менялись маршруты, права, экраны, операции или инварианты (см. правило в `93_GURU_GIT_COMMIT_STANDARD.md`).
8. **Тесты** добавлены для новой логики или в отчёте явно указано, почему нет (и заведён tech debt).
9. **`php artisan test`** проходит в `backend/`.
10. **`flutter analyze`** проходит в `mobile_app/` (после при необходимости `flutter gen-l10n`).
11. **`git add`** только точечно по путям; не `git add .` без осознанной необходимости.
12. **`git push`** только по явной просьбе в задаче.
13. В описании PR/отчёте агента указаны **ограничения**, **известный техдолг** и что сознательно не делалось.
